PHASE 4 Cross-device real-time E2E ★

The headline test: proves SignalR + Kafka + the order pipeline across all 4 devices. Watch propagation latency — should be near-instant (< 2s).

★ Scenario A — Dine-in order lifecycle (all 4 devices)

  1. Phone-1 (Customer) scans /qr for a table → orders 3 items → pays.
  2. PC-1 (KDS) ticket appears automatically (OrderPlaced).
  3. Phone-2 (Waiter) sees the table go active (TableUpdated/OrderPlaced).
  4. PC-1 chef marks items ready → Phone-2 gets OrderReady; Phone-1 /orders/[id] flips to READY.
  5. PC-2 (POS) cashier settles the check → table frees on Phone-2.

Scenario B — Delivery → Courier

  1. Phone-1 customer places a delivery order + pays.
  2. PC-1 KDS prepares → READY.
  3. Phone-2 (Courier) sees it in available list → accepts → checkpoints → delivered.
  4. Customer /delivery/[id] tracks status live.

Scenario C — Call Waiter

  1. Phone-1 customer taps “call waiter” → Phone-2 Waiter gets CallWaiter toast within ~1s.

Scenario D — Reservation

  1. Phone-1 books a table → PC-2 Admin/Waiter sees it; cancel reflects on both.

Scenario E — Notifications & loyalty

  1. Any qualifying action (order placed, bonus earned) → Notify bell badge increments on the customer device in real time.

Scenario F — Concurrency

  1. Two customers (Phone-1 + a PC tab) order the last unit of a low-stock item simultaneously → exactly one succeeds; the other sees sold-out.
