Core Framework

The runtime foundation — routing, rendering, server-side route protection and the BFF (backend-for-frontend) API layer.

N

Next.js ^15.1.0 App Router

React meta-framework providing file-based routing, server components, edge middleware and serverless API routes.
Why here: Tablero has eight role-based areas (/admin, /kds, /pos, /waiter, /courier, consumer). The App Router's route groups ((consumer)) and Edge Middleware let the app gate every URL by role before a page renders, and its API routes hold the JWT in httpOnly cookies so tokens never touch localStorage.
System-triggered
A CHEF requests /admin/revenue
middleware.js runs at the edge, reads the tablero_role cookie, sees the role isn't allowed, and redirects to that role's home (/kds) — no admin code is ever shipped to the browser.
User-triggered
A guest visits /checkout while logged out
The matcher catches the auth-required path and bounces them to /login?next=/checkout, so after signing in they land exactly where they intended. 
// middleware.js — server-side role gate (runs before render)
for (const { prefix, roles } of ROLE_GATES) {
  if (pathname.startsWith(prefix)) {
    if (!auth) return loginRedirect(request, pathname);
    if (!roles.includes(role))
      return NextResponse.redirect(new URL(ROLE_HOME[role], request.url));
  }
}

React ^19.0.0

The component & state library underneath everything; hooks drive all client-side interactivity.
Why here: The whole interface is component-driven. A hand-rolled Context store (lib/store.js) keeps the signed-in user, cart, notifications and preferences in one place, hydrated asynchronously from cookies on mount — no heavyweight state library needed.
System-triggered
App boots
A useEffect in StoreProvider restores the session from httpOnly cookies, decodes the JWT into a user object, and flips hydrated:true — components swap skeletons for real data.
User-triggered
Customer taps "Add to cart"
store.addToCart(item) updates context immutably; if the item already exists its qty increments, and every subscribed component re-renders instantly.

Design System & Styling

A bespoke "warm restaurant" visual language — terracotta & cream tokens, four typefaces, and a dark "kitchen display" theme.

T

Tailwind CSS ^3.4.17

Utility-first CSS engine, extended with the project's custom color/typography design tokens.
Why here: Tablero needs two coherent palettes — a warm cream UI for customers and a dark, high-contrast kds palette for the kitchen screen. Tailwind's theme.extend defines semantic tokens (terracotta, cream, ink, kds.bg, ok/warn/err) driven by CSS variables, so darkMode:'class' flips the whole app.
System-triggered
A kitchen ticket ages past 5 min
The card border swaps to a token class — border-t-err with animate-pulse-soft — turning the ticket red so the line cook can't miss it.
User-triggered
User enables dark mode in preferences
ThemeApplier toggles the .dark class on <html>; every hsl(var(--…)) token re-resolves and the palette inverts.

Data Layer & Forms

How the frontend talks to the ASP.NET backend, validates input, and formats time.

⇄

Axios ^1.8.4

HTTP client wired with interceptors for JWT auth, silent token refresh and C#→JS key normalization.
Why here: The C# gateway returns PascalCase JSON and protects routes with short-lived JWTs. Axios interceptors centralise three concerns once: attach Bearer tokens, transparently refresh on 401 and retry, and recursively lowercase every key to camelCase so React code stays idiomatic.
System-triggered
An access token expires mid-session
The response interceptor catches the 401, calls /api/auth/refresh once (deduped across concurrent calls), updates the token and replays the original request — the user notices nothing.
User-triggered
User submits the login form
api.post('/api/auth/login', {…}) fires, the response is unwrapped & camelCased, and store.login() persists the tokens to httpOnly cookies. 
// lib/client.js — refresh-once-and-retry on 401
http.interceptors.response.use(res => res, async err => {
  if (err.response?.status !== 401 || err.config._retry) throw err;
  err.config._retry = true;
  refreshing ??= fetch('/api/auth/refresh', { method: 'POST' })...
  const token = await refreshing;
  return http(err.config); // replay original request
});

Forms & Validation

react-hook-form + @hookform/resolvers + zod for typed schema validation; date-fns for date math.
Why here: Forms are everywhere — login, multi-step registration, checkout, staff/menu CRUD, reviews. Zod declares the validation rules once as a schema, react-hook-form wires them to inputs with minimal re-renders, and the resolver bridges the two. date-fns handles reservation/delivery date formatting.
System-triggered
Server rejects a registration
The backend's field errors map back onto the form: Object.values(errData.errors).flat() is surfaced inline and via toast, so each invalid field is annotated.
User-triggered
User blurs an empty password field
Validation runs immediately and sets errs.password = t('errPassword'), blocking submit until the field is filled — no round-trip needed.

Image Upload FormData · multipart

Menu-photo uploads sent as native FormData through the Axios client to the Cloudinary-backed media endpoint — no extra upload library.
Why here: Managers attach a photo to each dish. Rather than ship a heavyweight uploader, the menu editor builds a plain FormData and lets Axios post it as multipart/form-data (boundary set automatically) to the gateway, which signs & stores the file on Cloudinary server-side and returns a permanent CDN URL. The browser only ever holds the file and the resulting url; the shared request interceptor still attaches the Bearer token, and a 5 MB client-side guard rejects oversized images before any bytes leave the device.
System-triggered
The upload resolves
The response is key-normalized to { url } and written into the item form's img field, so the modal preview swaps from placeholder to the live Cloudinary thumbnail — and a 503 instead surfaces uploadNotConfigured as a toast.
User-triggered
Manager picks a dish photo
The file input calls uploadImage(file); it rejects anything over 5 MB with toast.error, else appends the file to a FormData and posts it to /api/restaurants/{id}/media/upload. 
// app/admin/menu — multipart upload via the shared Axios client
const fd = new FormData();
fd.append('file', file);                       // raw image, no JSON
const { url } = await api.post(`/api/restaurants/${restaurantId}/media/upload`, fd);
setForm(f => ({ ...f, img: url }));            // store the returned CDN URL

Real-Time & Payments

The features that make Tablero feel live — push events from the kitchen and the road, charts, and secure card payments.

⚡

SignalR @microsoft/signalr ^10

WebSocket real-time client for the ASP.NET Core hubs — kitchen, waiter, table, notify, cashier, courier.
Why here: A kitchen display and a courier map must update the instant the backend changes — polling is too slow and too chatty. SignalR keeps a JWT-authenticated socket open with automatic reconnect (back-off 0,2s,5s,10s,30s) and gracefully degrades to 15-second polling if a hub isn't reachable, so screens never go stale.
System-triggered
Backend emits OrderPlaced
The KDS hub fires conn.on('OrderPlaced', () => loadQueue()); the new ticket pops onto the board and a chime plays — no refresh, no click.
User-triggered
Chef opens the kitchen screen
On mount the page connects and calls conn.invoke('JoinGroup', 'kitchen:'+restaurantId), subscribing only to that restaurant's order stream. 
// lib/signalr.js — JWT auth + resilient reconnect
new signalR.HubConnectionBuilder()
  .withUrl(url, { accessTokenFactory: () => tokenStore.getAccess() })
  .withAutomaticReconnect([0, 2000, 5000, 10000, 30000])
  .build();
// app/kds — fall back to polling if the socket is down
if (!hubConnected) setInterval(loadQueue, 15000);

Internationalization & Theming

One codebase, six languages including right-to-left Arabic, plus persisted dark mode.

文

next-intl ^4.13.0 & next-themes ^0.4.6

Locale-aware translations (useTranslations) for 6 languages with RTL support, plus theme management.
Why here: Tablero targets a multilingual market — English, Russian, Azerbaijani, Turkish, Arabic and Chinese (~1,200 keys each). next-intl loads only the active locale's messages on the server, supports rich placeholders (t('minutesAgo', {n})), and the root layout flips dir="rtl" for Arabic. next-themes / a custom ThemeApplier persists dark mode.
System-triggered
A request arrives with NEXT_LOCALE=ar
i18n/request.js loads messages/ar.json server-side and the layout sets dir="rtl", so the entire app mirrors for Arabic readers.
User-triggered
User picks "Türkçe" in the LanguageSwitcher
The NEXT_LOCALE cookie updates and every useTranslations() string — including pluralised relative times — re-renders in Turkish. 
// i18n/request.js — load only the active locale, server-side
const raw = cookieStore.get('NEXT_LOCALE')?.value ?? 'en';
const locale = SUPPORTED.includes(raw) ? raw : 'en';
return { locale, messages: (await import(`../messages/${locale}.json`)).default };
